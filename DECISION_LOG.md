# 문장군 블로그 결정 로그

> 목적: 채팅방이 바뀌어도 중요한 결정의 이유와 영향 범위를 잃지 않기 위한 장기 기록.
> `_context.md`는 현재 상태용, 이 파일은 누적 결정 보관용이다.
> 인스타그램 관련 결정은 `문장군 인스타그램/DECISION_LOG.md`에서 관리한다.

## 문서 우선순위

충돌이 생기면 아래 순서로 판단한다.

1. `AGENTS.md` — 프로젝트 헌법, 브랜드/금지사항/실행 트리거
2. `DECISION_LOG.md` — 장기 결정과 변경 이유
3. `PRD_*.md` — 제품·기능 요구사항
4. `BRAND_CONTEXT.md`, `CONTENT_PLAN.md` — 브랜드·발행 기준
5. 외부 `SKILL.md` — 실행 방법론 (블로그엔진)
6. `_context.md` — 현재 작업 상태
7. 채팅 내용 — 즉시 맥락. 장기 결정은 반드시 파일로 승격

## 결정 기록

## DEC-017: ⛔ 현관문 교체 콘텐츠는 블로그 전략에서 완전 제외 — 2026-05-18
- **배경:** SEO_KEYWORD_RESEARCH.md 클러스터 E(현관문교체/현관문가격 등)에 키워드가 존재하지만, 사장님이 현관문 관련 블로그 콘텐츠를 작성하지 않겠다고 명시적으로 결정함.
- **결정:** 현관문 관련 키워드(아파트현관문교체, 현관문교체, 현관문교체비용, 현관문가격, 단열현관문) 일체를 CONTENT_PLAN에 포함하지 않는다. 향후 Phase에서도 제안 금지.
- **영향:** `CONTENT_PLAN.md`, `SEO_KEYWORD_RESEARCH.md`(클러스터 E 비활성), 모든 신규 콘텐츠 기획
- **재검토 조건:** 사장님이 직접 현관문 콘텐츠를 재개하겠다고 결정할 때만

## DEC-018: 알고리즘 변동 대응 — 지역 시공사례 확장 (Phase 4) — 2026-05-18
- **배경:** 2026-05-18 기준 네이버 블로그탭 알고리즘 대규모 이상 발생 (선거철 추정). 추적 50개 키워드 중 14개가 TOP30 밖으로 이탈.
- **결정:**
  1. 리라이팅은 알고리즘 정상화 후로 보류
  2. 하루 1편 발행 유지 (블로그 활동 지수 방어)
  3. Phase 4 = 지역 시공사례 대량 확장(청주/대전/평택/세종/수원/용인/안산 등) + 미작성 중급 키워드 3개
- **근거 문서:** `PHASE4_ANALYSIS_20260518.md`
- **영향:** `CONTENT_PLAN.md`(v4.0 Phase 4 추가), `POSTING_REGISTRY.md`
- **재검토 조건:** 알고리즘 정상화 확인 후 리라이팅 대상 확정 시. 또는 지역 글 10편 발행 후 전환 성과 평가 시.

## DEC-019: 인스타그램 프로젝트 완전 분리 — 2026-05-18
- **배경:** 블로그 프로젝트에 인스타그램 시스템이 혼재하면서 도메인 경계 붕괴. 에이전트 역할 불명확, 컨텍스트 비대화 발생.
- **결정:** 인스타그램을 `문장군 인스타그램` 독립 프로젝트로 완전 분리. 블로그 프로젝트는 순수 SEO 파이프라인으로 복원.
- **영향:** `AGENTS.md`(인스타 섹션 제거), `GEMINI.md`(업데이트), `DECISION_LOG.md`(분리)
- **재검토 조건:** 불필요. 이 결정은 영구적.

## 사용 규칙

- 중요한 방향 전환, 저장 규칙, KPI, 금지사항 변경은 반드시 이 파일에 추가한다.
- `_context.md`에는 최신 5개만 남기고, 장기 보존은 이 파일을 기준으로 한다.
- 인스타그램 관련 결정은 이 파일에 기록하지 않는다.
