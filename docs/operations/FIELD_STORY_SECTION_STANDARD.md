# 현장 단락 고정 기준

> 목적: 문장군 블로그 글이 단순 설명형으로 마르지 않도록 101번 이후 신규 현장형 원고의 본문 구조를 고정한다.
> 최종 업데이트: 2026-06-24

## 1. 필수 단락

101번 이후 신규 현장형 원고는 본문 안에 아래 H2 단락을 반드시 포함한다.

```markdown
## 실제 시공 현장에서는 조금 다릅니다
```

이 단락이 없으면 `scripts/validate_post.js`가 FAIL로 처리하며, 네이버 발행 금지다.

## 2. 단락 목적

이 단락은 제품 설명을 현장 판단으로 바꾸는 장치다.

검색자는 보통 가격, 종류, 단점, 소음, 유리, 간섭처럼 단편적인 키워드로 들어온다. 문장군 글은 그 키워드를 그대로 반복하지 않고, 실제 현장에서 무엇이 달라지는지 보여줘야 한다.

## 3. 권장 흐름

필수 단락은 아래 흐름을 따른다.

```text
고객 고민 또는 처음 생각
→ 현장에서 확인한 구조 변수
→ 문장군이 판단한 기준
→ 진행 방향이 정해진 이유
→ 시공 후 변화 또는 상담 전 준비 기준
```

이 흐름은 AppSheet 현장 치환을 위한 슬롯이다. 직원이 실제 현장을 찾으면 첫 문장을 `노원구 상계주공아파트 현장도 비슷했습니다`처럼 바꾸고, 이어서 비포 사진, 실측 사진, 완료 사진을 붙여도 글이 자연스럽게 이어져야 한다.

필수 단락에는 아래 요소가 자연문으로 들어가는 것이 좋다.

| 요소 | 목적 |
| --- | --- |
| 지역/아파트/구조로 치환 가능한 첫 문장 | AppSheet 검색과 실제 현장명 삽입 |
| 고객이 망설인 이유 | 사건성, 자기 연관성 |
| 현장에서 본 변수 | 사진 매칭 기준 |
| 문장군의 판단 | 전문성 |
| 진행 방향 또는 시공 후 변화 | 결과와 CTA 연결 |

## 4. 예시 형식

```markdown
## 실제 시공 현장에서는 조금 다릅니다

실제로 비슷한 구축 아파트 현장에서도 처음에는 원슬라이딩 중문을 보고 상담을 시작하신 경우가 있었습니다.

사진상으로는 벽면이 충분해 보였지만, 현장에서 신발장 문을 직접 열어보니 중문 프레임이 들어갈 위치와 문 열림 범위가 겹쳤습니다.

이 현장은 단순히 중문 종류만 바꾸는 문제가 아니었습니다. 신발장 간섭, 손잡이 돌출, 스위치 위치, 바닥 레일 위치까지 같이 봐야 했습니다.

정리하면 좁은 현관에 맞는 중문은 사진보다 실제 동작감이 더 중요합니다. 신발장 문을 열어보고 프레임이 앉을 선을 봐야 중문 종류가 제대로 정해집니다.
```

## 5. 증거와 표현 기준

- AppSheet 또는 등록부상 근거 축이 있으면 지역/구조/제품을 익명화해 현장 단락으로 쓴다.
- 고객명, 주소, 상담 원문, 원본 사진 ID는 공개 저장소에 남기지 않는다.
- `evidence_ref`가 없으면 직접 인용문이나 실제 고객 감정처럼 단정하지 않는다.
- 근거가 아직 확정되지 않은 경우에도 현장 단락은 쓴다. 단, 실제 고객 사례처럼 꾸미지 않고 `비슷한 현장`, `현장에서 자주 확인되는 조건`처럼 일반화한다.
- 실제 실패 사진이 없으면 실패 사례로 쓰지 않는다.
- 실패 여부를 말하려면 실제 재시공, 철거, 하자 사진을 붙일 수 있는 근거가 있어야 한다.

## 6. 금지하는 작성 방식

아래 방식은 금지한다.

- 설명만 이어지고 현장 단락이 없는 글
- "현장에서 자주 봅니다"만 있고 구체 구조 변수가 없는 글
- AppSheet 사진을 붙이기 어려운 가공 사례
- 실제 고객처럼 보이지만 근거가 없는 대화체
- 실패 사진이 없는데 실패 현장처럼 꾸민 글
- 무료 실측을 셀프 시공 권유처럼 연결하는 CTA

## 7. 제작노트 금지, 본문형 사진 유도

별도 제작노트 파일은 만들지 않는다.

`outputs/drafts/NNN_키워드_note.md` 방식은 폐기한다. 사장님과 직원이 보지 않는 파일은 운영상 없는 파일과 같고, 원고와 사진 기준이 분리되면 현장형 글맛이 흔들린다.

대신 발행 본문 자체가 사진을 부르도록 쓴다.

```markdown
현관 전체가 넓어 보여도 신발장 문을 실제로 열어보면 중문 프레임이 들어갈 선과 겹치는 경우가 있습니다.

그래서 상담 전 사진을 찍을 때는 현관 정면 사진뿐 아니라 신발장 문을 연 상태, 스위치 위치, 바닥 레일이 들어갈 선까지 함께 확인하는 것이 좋습니다.
```

이 문장만으로 직원은 현관 전체, 신발장 문 열린 상태, 스위치 위치, 바닥선을 보여주는 사진을 찾을 수 있다.

발행 원고 안에 `[사진:]`, `[이미지:]`, `[AppSheet 확인]`, `[제작자 메모]`, `## 운영 메모` 같은 내부 지시문을 남기면 발행 불가다.
