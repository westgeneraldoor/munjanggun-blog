# 문장군 블로그 데이터 보안 정책

> 최종 업데이트: 2026-06-18  
> 목적: 공개 저장소에 네이버 관리자 통계 원본, 고객 자료, 내부 성과 원본이 섞이지 않도록 막는다.

## 1. 저장소에 올릴 수 없는 자료

아래 자료는 Git에 커밋하지 않는다.

- 네이버 블로그 관리자 통계 원본 XLSX
- 네이버 블로그 관리자 화면 스크린샷
- 원본 유입경로 이미지
- 고객명, 전화번호, 주소, 예약번호, 상담 원문
- AppSheet 원본 레코드, 리뷰 원본, 현장 사진 원본
- 비공개 매출, 광고, 계정, 토큰, 쿠키, 세션 정보

`.gitignore` 기준:

```gitignore
data/naver/daily/
data/naver/raw/
data/naver/**/*.xlsx
```

## 2. 저장소에 남길 수 있는 자료

아래처럼 비식별·요약된 결과만 저장한다.

- `outputs/reports/daily/YYYY-MM-DD_seo_watch.md`
- 날짜별 조회수, 공감수, 댓글수, 이웃증감수
- 주요 유입경로 비율
- 주요 검색어와 비율
- 게시글 TOP 20 제목과 조회수
- 총괄 판단, 다음 액션

검색어는 공개 검색어 수준으로만 남긴다. 고객 개인을 식별할 수 있는 검색어, 예약자명, 주소, 전화번호가 포함되면 제거한다.

## 3. 고객 사례 evidence_ref 규칙

공개 저장소에는 실제 고객 자료를 넣지 않는다. 원고나 제작 노트에는 비공개 자료를 가리키는 불투명 ID만 남긴다.

허용 예:

```json
{
  "source_type": "appsheet_case",
  "evidence_refs": ["APPSHEET_CASE_20260613_001"],
  "quote_status": "paraphrased",
  "privacy_status": "anonymized"
}
```

금지 예:

```json
{
  "customer_name": "홍길동",
  "phone": "010-0000-0000",
  "address": "..."
}
```

## 4. 원본 자료 보관 위치

원본 자료는 AppSheet, 로컬 보안 폴더, 또는 접근 권한이 제한된 비공개 저장소에서 관리한다. GitHub 공개 저장소에는 올리지 않는다.

## 5. 노출 사고 대응 순서

공개 저장소에 원본 자료가 올라간 사실을 확인하면 아래 순서로 처리한다.

1. 저장소를 Private으로 전환한다.
2. 원본 파일에 개인정보·계정정보·비공개 성과 정보가 있는지 확인한다.
3. 민감 자료라면 현재 브랜치뿐 아니라 Git 이력에서도 제거한다.
4. PR 브랜치를 정리된 이력으로 강제 갱신한다.
5. `.gitignore`에 경로 단위 차단 규칙을 추가한다.
6. 공개 전 `git ls-files`로 원본 파일이 남지 않았는지 확인한다.

## 6. 공개 전 확인 명령

```bash
git ls-files "data/naver/daily/**" "data/naver/raw/**" "*.xlsx"
git status --ignored --short data/naver
```

첫 번째 명령 결과가 비어 있어야 한다.
