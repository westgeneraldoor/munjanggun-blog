# 문장군 블로그 주제 선정 스코어카드 v1.0

> 목적: 신규 포스팅 주제를 "좋아 보이는 감"이 아니라 시장 수요, 실제 반응, 문장군 현장성, 발행 안전성을 함께 본 운영 판단으로 선정한다.

## 핵심 원칙

포스팅 주제는 블로그 유입어만으로 선정하지 않는다.

네이버 광고 API로 시장 전체 수요를 먼저 확인하고, 네이버 블로그 통계로 문장군 블로그의 실제 반응을 검증한 뒤, 문장군 서비스 적합성, AppSheet 현장 연결성, 기존 글과의 중복 여부, 발행 안전성을 통과한 주제만 발행 후보로 삼는다.

```text
네이버 광고 API = 시장 전체 지도
네이버 블로그 통계 = 문장군 블로그의 실제 발자국
게시글 TOP20 = 먹히는 글 구조
브랜드 컨텍스트 = 할 수 있는 말 / 하면 안 되는 말
AppSheet = 실제 현장성과 사진 연결성
```

## 3단계 판단 구조

### 1단계: 시장 지도

네이버 검색광고 API 기준으로 시장 전체 검색 수요를 본다.

확인 항목:

- 월간 검색량
- 모바일 검색 비중
- 경쟁 강도
- 연관 키워드 확장성
- 허브 키워드인지, 클러스터 키워드인지, 롱테일 키워드인지

운영 명령:

```powershell
npm run keywords:regional
npm run keywords:product
```

로컬 `package.json`에 스크립트가 없거나 공개 레포와 차이가 있으면 로컬 최신 프로젝트를 기준으로 한다. 필요하면 아래 직접 실행도 허용한다.

```powershell
node scripts/fetch_keyword_data.js
node scripts/fetch_keyword_data_product.js
```

시장 지도에서 대표적으로 보는 축:

- 중문
- 3연동중문
- 현관중문
- 방문교체
- ABS도어
- 문짝교체비용
- 아파트 중문 설치비용
- 중문설치비용
- 중문 유리종류

### 2단계: 실전 반응

네이버 블로그 통계 기준으로 문장군 블로그에 실제로 들어온 반응을 본다.

확인 항목:

- 실제 유입어 반복 여부
- 신규 글의 게시글 TOP20 진입 여부
- 오래 살아남는 게시글 유형
- 문장군 블로그가 이미 먹고 있는 검색 의도
- 최근 상승 중인 키워드

예시:

- 플루트유리
- 모루유리 중문
- 좁은 현관 중문
- 문짝교체비용
- 셀프중문
- 12mm 문선
- 거주중 중문시공

### 3단계: 문장군 필터

검색량이 좋아도 문장군 기준에 맞지 않으면 발행 후보에서 제외하거나 안전한 방향으로 전환한다.

| 키워드/상황 | 판단 |
| --- | --- |
| 현관문 | DEC-017 기준 영구 제외 |
| 방화문 | DEC-017 기준 영구 제외 |
| 문틀만 교체 | 문장군 운영 기준상 오해 유발 가능. 문짝+문틀세트, 문짝교체+문틀필름, 쌈지마감 등으로 전환 |
| 셀프중문 | 셀프 권장이 아니라 실패 방지, 자재판매 미진행, 전문 실측 필요성으로 전환 |
| 9mm 문선 | 문장군 상품처럼 쓰지 않는다. 12mm 슬림문선 비교/전환 글로만 사용 |
| 비대칭양개형중문 | 문장군 미취급. 제외 |
| 중문파티션 | 문장군 미취급. 제외 |
| 신발장리폼 | 제외 아님. 중문 설치 시 신발장 돌출, 간섭, 측면 가공, 추가금 변수로 사용 가능 |
| 문짝교체비용 | 가능. 문짝만 교체, 문짝+문틀세트, 문틀필름, 쌈지마감 구분 필요 |

## 주제 선정 점수표

후보 주제는 아래 비중으로 판단한다.

| 기준 | 비중 | 판단 의미 |
| --- | ---: | --- |
| 네이버 광고 API 시장 수요 | 25 | 전체 네이버에서 검색량이 충분한가 |
| 블로그 실제 유입 반복성 | 20 | 문장군 블로그에 이미 들어오고 있는가 |
| 게시글 TOP20 반응 | 15 | 비슷한 글 구조가 이미 먹혔는가 |
| 문장군 서비스 적합성 | 15 | 실제 취급/시공 가능한 주제인가 |
| AppSheet 현장 연결성 | 10 | 사진, 사례, Before/After 붙이기 쉬운가 |
| 기존 글과 중복/카니발 방지 | 10 | 기존 글과 겹치지 않는가 |
| 발행 안전성 | 5 | 과장, 불가 서비스, 허위 사례 위험이 없는가 |

총점은 100점이다.

판정 기준:

| 총점 | 판정 |
| ---: | --- |
| 80점 이상 | 바로 작성 후보 |
| 65~79점 | 보정 후 작성 후보 |
| 50~64점 | 관찰 또는 내부링크/리라이팅 후보 |
| 49점 이하 | 보류 또는 제외 |

## 키워드 유형별 처리

| 유형 | 처리 |
| --- | --- |
| 광고 API 검색량 높음 + 블로그 유입도 반복됨 | 바로 발행 1순위 |
| 광고 API 검색량 높음 + 블로그 유입은 아직 약함 | 신규 공략/허브 글 후보 |
| 광고 API 검색량 낮음 + 블로그 유입 반복됨 | 문장군 맞춤 롱테일 전환 글 |
| 광고 API 검색량 높음 + 문장군이 안 하는 서비스 | 제외 또는 안전하게 방향 전환 |
| 블로그 유입 있음 + AppSheet 사진 연결 어려움 | 보류 또는 일반 정보글로 축소 |
| 기존 글과 거의 겹침 | 제목만 바꾸지 말고 관점 재설계 |

## 후보 보고 형식

신규 글감 또는 다음 포스팅 후보를 보고할 때는 아래 형식을 사용한다.

운영 산출물은 아래 위치에 저장한다.

```text
outputs/reports/topic_candidates/YYYY-MM-DD_topic_scorecard.md
```

템플릿은 `outputs/reports/topic_candidates/TOPIC_SCORECARD_TEMPLATE.md`를 사용한다.

```text
후보 1. 주제명
- 광고 API 시장 수요:
- 블로그 실제 유입:
- TOP20 관련 반응:
- 문장군 서비스 적합성:
- AppSheet 사진/현장 연결성:
- 기존 글 중복 여부:
- 발행 안전성:
- 총괄 판정:
```

예시:

```text
후보 1. 중문 유리 종류
- 광고 API 시장 수요: 중문/3연동중문/중문 유리종류 계열 확장 가능
- 블로그 실제 유입: 플루트유리, 모루유리 중문, 3연동 중문 유리종류 반복
- TOP20 관련 반응: 091_3연동중문모루유리 진입
- 문장군 서비스 적합성: 가능
- AppSheet 사진/현장 연결성: 유리 샘플, 현관 Before, 시공 완료 사진 연결 쉬움
- 기존 글 중복 여부: 091과 겹치지 않게 유리 선택 기준/현장 구조 관점으로 분화
- 발행 안전성: 보장성 표현 없이 선택 기준으로 작성 가능
- 총괄 판정: 작성 후보
```

## 운영 주기

매일:

- 네이버 블로그 실제 유입어와 게시글 TOP20을 기록한다.
- 단일 날짜 반짝 검색어만으로 방향을 고정하지 않는다.
- 신규 글감 후보를 제안했다면 topic scorecard를 남긴다. 현재 `ops:daily`에서는 누락을 WARN으로 표시한다.

매주:

- 네이버 광고 API 키워드 데이터를 확인한다.
- 블로그 실제 유입어와 매칭해 신규 공략 후보를 3~5개로 압축한다.
- 검색량은 크지만 문장군 미취급인 키워드는 제외/전환 목록에 반영한다.

매월:

- `docs/strategy/SEO_KEYWORD_RESEARCH.md`와 `docs/strategy/CONTENT_PLAN.md`를 재검토한다.
- 허브, 클러스터, 롱테일 구조를 다시 정리한다.

## 주의

- 광고 API 검색량이 높다는 이유만으로 발행하지 않는다.
- 실제 블로그 유입이 반복된다는 이유만으로 시장성이 충분하다고 판단하지 않는다.
- AppSheet 현장을 먼저 찾느라 글 작성을 멈추지 않는다. 다만 원고에는 실제 현장으로 치환 가능한 `실제 시공 현장` 단락을 반드시 설계한다.
- 공개 저장소에는 네이버 관리자 원본 통계, XLSX, 화면 캡처, 고객 원본 자료를 커밋하지 않는다.
- 최종 발행 전에는 `scripts/blog_quality_gate.js` 기준을 통과해야 한다.
