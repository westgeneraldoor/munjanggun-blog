# 문장군 블로그 주제 선정 스코어카드 v1.1

> 목적: 신규 포스팅 주제를 "좋아 보이는 감"이 아니라 시장 수요, 실제 반응, 고객 불안, 문장군 현장성, 발행 안전성을 함께 본 운영 판단으로 선정한다.

> 2026-07-01 claim gate: 상품 위키는 구조/선택 기준/이미지 연결을 보는 문서이고, 가격·리뷰 수·A/S·일정·지역·이벤트·배송/반품·패키지 구성 claim은 중앙 `EVIDENCE_REGISTER.md`와 `OPEN_QUESTIONS_REGISTER.md` 상태를 함께 확인해야 한다. `candidate`, `open`, `in_review` 상태는 발행 문장으로 단정하지 않는다.

## 핵심 원칙

포스팅 주제는 블로그 유입어만으로 선정하지 않는다.

네이버 광고 API로 시장 전체 수요를 먼저 확인하고, 네이버 블로그 통계로 문장군 블로그의 실제 반응을 검증한 뒤, 문장군 서비스 적합성, AppSheet 현장 연결성, 기존 글과의 중복 여부, 발행 안전성을 통과한 주제만 발행 후보로 삼는다.

키워드는 검색 유입의 근본이다. 다만 키워드명을 그대로 제목으로 옮기는 것은 글감이 아니다. 문장군 신규 글감은 아래 순서를 따른다.

```text
시장 키워드/실제 유입어
→ 고객 상황
→ 고객 불안·의심 문장
→ 제목 후보
→ 문장군 취급 가능성/중복/게이트 검증
```

예시:

```text
키워드: 중문설치, 아파트중문, 전세집 중문
고객 상황: 전세집이라 벽이나 바닥 손상이 걱정됨
고객 불안: 전세집에 중문 설치해도 괜찮을까?
제목화: 전세집 중문 설치 전 집주인에게 확인할 3가지
```

검색 키워드는 제목과 도입부에 자연스럽게 반영한다. 그러나 제목의 표면은 고객이 검색창에 치기 직전의 말, 즉 불안·의심·상황에 가까워야 한다.

```text
네이버 광고 API = 시장 전체 지도
네이버 블로그 통계 = 문장군 블로그의 실제 발자국
게시글 TOP20 = 먹히는 글 구조
브랜드 컨텍스트 = 할 수 있는 말 / 하면 안 되는 말
AppSheet = 실제 현장성과 사진 연결성
```

## 3단계 판단 구조

### 1단계: 시장 지도

네이버 검색광고 API 기준으로 시장 전체 검색 수요를 본다.

확인 항목:

- 월간 검색량
- 모바일 검색 비중
- 경쟁 강도
- 연관 키워드 확장성
- 허브 키워드인지, 클러스터 키워드인지, 롱테일 키워드인지

운영 명령:

```powershell
npm run keywords:regional
npm run keywords:product
```

로컬 `package.json`에 스크립트가 없거나 공개 레포와 차이가 있으면 로컬 최신 프로젝트를 기준으로 한다. 필요하면 아래 직접 실행도 허용한다.

```powershell
node scripts/fetch_keyword_data.js
node scripts/fetch_keyword_data_product.js
```

시장 지도에서 대표적으로 보는 축:

- 중문
- 3연동중문
- 현관중문
- 방문교체
- ABS도어
- 문짝교체비용
- 아파트 중문 설치비용
- 중문설치비용
- 중문 유리종류

### 1-1단계: 고객 문장화

시장 키워드를 확인한 뒤 바로 제목을 만들지 않는다. 먼저 그 키워드를 검색하는 고객의 상황과 불안을 문장으로 바꾼다.

확인 항목:

- 이 키워드를 검색하는 고객은 어떤 집 상태에 놓여 있는가
- 비용, 손상, 집주인, 이사, 효과, 타이밍, 추가금 중 무엇을 걱정하는가
- 고객이 실제로 할 법한 말로 바꾸면 어떤 문장인가
- 이 문장이 문장군 무료 방문실측이나 상담 흐름으로 자연스럽게 이어지는가
- 제목이 기존 글과 독자 눈에 다르게 보이는가

좋은 출발 문장:

```text
전세집에 중문 설치해도 괜찮을까?
전체 인테리어는 부담인데 문짝만 교체해도 될까?
중문 설치해도 정말 효과가 있을까?
이사 가는 집에 중문이 없으면 꼭 해야 할까?
몰딩시공은 언제 해야 돈이 덜 들까?
```

나쁜 출발 문장:

```text
중문시공 기준
모루유리중문 선택 기준
문틀시트지 시공 기준
```

위 문장들은 키워드일 수는 있지만 고객의 상황이 약하다. 글감 후보로 올리려면 고객 불안 문장으로 다시 바꾼다.

### 2단계: 실전 반응

네이버 블로그 통계 기준으로 문장군 블로그에 실제로 들어온 반응을 본다.

확인 항목:

- 실제 유입어 반복 여부
- 신규 글의 게시글 TOP20 진입 여부
- 오래 살아남는 게시글 유형
- 문장군 블로그가 이미 먹고 있는 검색 의도
- 최근 상승 중인 키워드

예시:

- 플루트유리
- 모루유리 중문
- 좁은 현관 중문
- 문짝교체비용
- 셀프중문
- 12mm 문선
- 거주중 중문시공

### 3단계: 문장군 필터

검색량이 좋아도 문장군 기준에 맞지 않으면 발행 후보에서 제외하거나 안전한 방향으로 전환한다.

| 키워드/상황 | 판단 |
| --- | --- |
| 현관문 | DEC-017 기준 영구 제외 |
| 방화문 | DEC-017 기준 영구 제외 |
| 문틀만 교체 | 문장군 운영 기준상 오해 유발 가능. 문짝+문틀세트, 문짝교체+문틀필름, 쌈지마감 등으로 전환 |
| 셀프중문 | 셀프 권장이 아니라 실패 방지, 자재판매 미진행, 전문 실측 필요성으로 전환 |
| 9mm 문선 | 문장군 상품처럼 쓰지 않는다. 12mm 슬림문선 비교/전환 글로만 사용 |
| 비대칭양개형중문 | 문장군 미취급. 제외 |
| 중문파티션 | 문장군 미취급. 제외 |
| 신발장리폼 | 제외 아님. 중문 설치 시 신발장 돌출, 간섭, 측면 가공, 추가금 변수로 사용 가능 |
| 문짝교체비용 | 가능. 문짝만 교체, 문짝+문틀세트, 문틀필름, 쌈지마감 구분 필요 |

## 주제 선정 점수표

후보 주제는 아래 비중으로 판단한다.

| 기준 | 비중 | 판단 의미 |
| --- | ---: | --- |
| 네이버 광고 API 시장 수요 | 20 | 전체 네이버에서 검색량이 충분한가 |
| 블로그 실제 유입 반복성 | 15 | 문장군 블로그에 이미 들어오고 있는가 |
| 게시글 TOP20 반응 | 10 | 비슷한 글 구조가 이미 먹혔는가 |
| 고객 불안/제목화 가능성 | 20 | 키워드를 고객 상황과 클릭 가능한 문장으로 바꿀 수 있는가 |
| 문장군 서비스 적합성 | 15 | 실제 취급/시공 가능한 주제인가 |
| AppSheet 현장 연결성 | 8 | 사진, 사례, Before/After 붙이기 쉬운가 |
| 기존 글과 중복/카니발 방지 | 7 | 기존 글과 겹치지 않는가 |
| 발행 안전성 | 5 | 과장, 불가 서비스, 허위 사례 위험이 없는가 |

총점은 100점이다.

판정 기준:

| 총점 | 판정 |
| ---: | --- |
| 80점 이상 | 바로 작성 후보 |
| 65~79점 | 보정 후 작성 후보 |
| 50~64점 | 관찰 또는 내부링크/리라이팅 후보 |
| 49점 이하 | 보류 또는 제외 |

## 키워드 유형별 처리

| 유형 | 처리 |
| --- | --- |
| 광고 API 검색량 높음 + 블로그 유입도 반복됨 | 바로 발행 1순위 |
| 광고 API 검색량 높음 + 블로그 유입은 아직 약함 | 신규 공략/허브 글 후보 |
| 광고 API 검색량 낮음 + 블로그 유입 반복됨 | 문장군 맞춤 롱테일 전환 글 |
| 광고 API 검색량 높음 + 문장군이 안 하는 서비스 | 제외 또는 안전하게 방향 전환 |
| 블로그 유입 있음 + AppSheet 사진 연결 어려움 | 보류 또는 일반 정보글로 축소 |
| 기존 글과 거의 겹침 | 제목만 바꾸지 말고 관점 재설계 |

## 포트폴리오 역할

후보마다 `portfolio_lane`과 `expected_role`을 명시한다. 이것은 글감의 좋고 나쁨만 보는 항목이 아니라, 일 2,000 조회수 목표 안에서 어떤 임무를 맡는지 판단하는 항목이다.

| portfolio_lane | expected_role | 판단 |
| --- | --- | --- |
| protect | 조회수 방어 | 이미 성과가 있는 키워드를 보호하고 내부링크/후속 글로 확장 |
| attack | 신규 유입 확대 | 시장 수요가 크지만 현재 블로그 성과가 약한 키워드 공략 |
| experiment | 전환 롱테일 | 검색량은 작아도 견적, 하자, 현장 불안, 교체 의도가 강한 키워드 실험 |
| exclude | 위험 차단 | 문장군 미취급, 오해 유발, 서비스 범위 밖 키워드 확장 금지 |

## 후보 보고 형식

신규 글감 또는 다음 포스팅 후보를 보고할 때는 아래 형식을 사용한다.

### 중앙 브랜드 확인 항목

각 후보는 아래 3가지를 함께 적는다.

- 상품 위키/선택 가이드 확인: 구조, 옵션, 이미지/GIF, 고객 선택 기준 확인 여부
- EVIDENCE_REGISTER claim 상태: 가격, 리뷰 수, A/S, 일정, 지역, 패키지 claim의 publishable/vetted 여부
- OPEN_QUESTIONS_REGISTER 미해결 여부: open/in_review claim이 제목이나 본문 단정으로 들어가지 않았는지

운영 산출물은 아래 위치에 저장한다.

```text
outputs/reports/topic_candidates/YYYY-MM-DD_topic_scorecard.md
```

템플릿은 `outputs/reports/topic_candidates/TOPIC_SCORECARD_TEMPLATE.md`를 사용한다.

```text
후보 1. 주제명
- portfolio_lane:
- expected_role:
- 광고 API 시장 수요:
- 블로그 실제 유입:
- TOP20 관련 반응:
- 고객 상황:
- 고객 불안/의심 문장:
- 제목화 방향:
- 문장군 서비스 적합성:
- AppSheet 사진/현장 연결성:
- 기존 글 중복 여부:
- 발행 안전성:
- 총괄 판정:
```

예시:

```text
후보 1. 중문 유리 종류
- portfolio_lane: attack
- expected_role: 신규 유입 확대
- 광고 API 시장 수요: 중문/3연동중문/중문 유리종류 계열 확장 가능
- 블로그 실제 유입: 플루트유리, 모루유리 중문, 3연동 중문 유리종류 반복
- TOP20 관련 반응: 091_3연동중문모루유리 진입
- 고객 상황: 유리 샘플은 예쁜데 우리 집 현관에서는 답답해 보일까 걱정함
- 고객 불안/의심 문장: 중문 유리, 사진처럼 예쁘게 나올까?
- 제목화 방향: 중문 유리 선택 전 우리 집 현관 밝기부터 보는 이유
- 문장군 서비스 적합성: 가능
- AppSheet 사진/현장 연결성: 유리 샘플, 현관 Before, 시공 완료 사진 연결 쉬움
- 기존 글 중복 여부: 091과 겹치지 않게 유리 선택 기준/현장 구조 관점으로 분화
- 발행 안전성: 보장성 표현 없이 선택 기준으로 작성 가능
- 총괄 판정: 작성 후보
```

## 운영 주기

매일:

- 네이버 블로그 실제 유입어와 게시글 TOP20을 기록한다.
- 단일 날짜 반짝 검색어만으로 방향을 고정하지 않는다.
- 신규 글감 후보를 제안했다면 topic scorecard를 남긴다. daily의 다음 액션은 `docs/strategy/ACTIVE_TOPIC_QUEUE.md`에 먼저 반영하고, scorecard가 필요한 후보는 `scorecard_needed`로 표시한다. 현재 `ops:daily`에서는 scorecard 누락을 WARN으로 표시한다.

매주:

- 네이버 광고 API 키워드 데이터를 확인한다.
- 블로그 실제 유입어와 매칭해 신규 공략 후보를 3~5개로 압축한다.
- 검색량은 크지만 문장군 미취급인 키워드는 제외/전환 목록에 반영한다.

매월:

- `docs/strategy/SEO_KEYWORD_RESEARCH.md`와 `docs/strategy/CONTENT_PLAN.md`를 재검토한다.
- 허브, 클러스터, 롱테일 구조를 다시 정리한다.

## 주의

- 광고 API 검색량이 높다는 이유만으로 발행하지 않는다.
- 실제 블로그 유입이 반복된다는 이유만으로 시장성이 충분하다고 판단하지 않는다.
- AppSheet 현장을 먼저 찾느라 글 작성을 멈추지 않는다. 다만 원고에는 실제 현장으로 치환 가능한 `실제 시공 현장` 단락을 반드시 설계한다.
- 공개 저장소에는 네이버 관리자 원본 통계, XLSX, 화면 캡처, 고객 원본 자료를 커밋하지 않는다.
- 최종 발행 전에는 `scripts/blog_quality_gate.js` 기준을 통과해야 한다.
