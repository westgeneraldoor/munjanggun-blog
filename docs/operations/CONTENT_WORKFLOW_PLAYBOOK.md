## 1. 세션 시작 체크

새 세션에서 블로그 작업을 시작하면 먼저 아래 파일을 순서대로 읽는다.

1. `AGENTS.md`
2. `docs/strategy/_context.md`
3. `docs/strategy/CONTENT_PLAN.md`
4. `docs/strategy/POSTING_REGISTRY.md`
5. `outputs/reports/ranking_report.md`
6. `outputs/reports/top10_analysis.md`
7. 필요 시 `docs/strategy/BRAND_CONTEXT.md`, `docs/strategy/SEO_KEYWORD_RESEARCH.md`

읽은 뒤 반드시 아래 4가지를 먼저 판단한다.

| 판단 항목 | 기준 |
| --- | --- |
| 신규 글 작성인지 | `CONTENT_PLAN.md`의 다음 미작성 슬롯이 명확한 경우 |
| 리라이팅인지 | 순위 하락, 클릭은 있으나 순위 약함, 브랜드 방어 필요가 있는 경우 |
| URL 등록인지 | `POSTING_REGISTRY.md`에 URL이 `-`로 남아 있고 사용자가 URL을 제공한 경우 |
| 분석/점검인지 | 사용자가 순위, 조회수, 유입어, 리라이팅 계획을 요청한 경우 |

## 5. 신규 글 작성 프로토콜

1. `CONTENT_PLAN.md`에서 다음 미작성 슬롯 확인
2. `POSTING_REGISTRY.md`에서 같은 키워드와 같은 소재가 이미 있는지 확인
3. `ranking_report.md`에서 관련 키워드 순위가 이미 강한지 확인
4. 강한 키워드는 직접 중복 작성하지 말고 롱테일/사례/비용/단점 각도로 분화
5. `top10_analysis.md`에서 제목 패턴 확인
6. 글 작성 후 발행 본문은 `posts/NNN_키워드.md`에 저장
7. 제목 후보, 검색 의도, 품질 채점, 예상 성능, 이미지 지시는 `outputs/drafts/NNN_키워드_note.md`에 저장
8. `npm run validate:posts`로 자동 검수하고 결과를 `outputs/checks/`에 저장
9. `CONTENT_PLAN.md` 상태는 원고완료 기준으로 업데이트
10. `POSTING_REGISTRY.md`에 파일, 제목, 소재 요약과 상태를 등록
11. 발행 전 `docs/operations/PREPUBLISH_CHECKLIST.md`를 확인
12. 발행 후 사용자가 URL을 주면 URL과 발행일 등록

### v2 산출물 분리 규칙

`posts/` 파일은 네이버 블로그에 그대로 붙여넣을 발행 본문만 담는다.

`posts/`에 남기는 것:
- 최종 제목
- 본문
- 관련 글 링크
- 태그란 입력용 해시태그

`outputs/drafts/`로 분리하는 것:
- 제목 후보
- 검색 의도 분석
- 썸네일 카피
- 품질 채점 결과
- 예상 성능
- 이미지 삽입 지시
- 발행 전 작업 메모

`posts/` 파일이 `# 제목 후보`로 시작하면 발행 불가로 본다.

## 6. 리라이팅 프로토콜

리라이팅은 무조건 아래 순서를 따른다.

1. 현재 순위와 하락 폭 확인
2. 네이버 통계 유입어가 있으면 함께 반영
3. 기존 글의 타겟 키워드, 제목, 도입부, 소제목, 내부링크를 점검
4. 기존 강점을 훼손하지 않는 보강안부터 제안
5. 제목 변경이 필요한 경우 기존 URL과 발행 이력 리스크를 명시
6. 리라이팅 원고는 반드시 새 번호의 새 파일로 저장한다
7. 기존 원본 파일은 절대 덮어쓰지 않는다
8. `POSTING_REGISTRY.md`에는 새 행으로 등록하고, 소재 요약에 원본 참조 번호를 남긴다

### 리라이팅 파일명 규칙

리라이팅은 기존 글 수정본이 아니라 새 발행본이다. 파일명은 아래 규칙을 강제한다.

```text
posts/NNN_키워드_리라이팅.md
```

예시:

```text
posts/068_문장군중문_리라이팅.md
posts/069_3연동중문_리라이팅.md
posts/070_아파트현관중문_리라이팅.md
```

금지:

```text
posts/002_3연동중문.md        # 기존 원본과 충돌
posts/002_3연동중문_RE.md     # 기존 번호 재사용
posts/3연동중문_리라이팅.md   # 번호 없음
```

`POSTING_REGISTRY.md` 등록 형식:

```text
| 069 | 069_3연동중문_리라이팅.md | 리라이팅(H2) | 3연동중문 | 새 제목 | - | - | 002_3연동중문.md의 순위 회복용 새 발행본. ... |
```

리라이팅 판단 기준:

| 상황 | 권장 작업 |
| --- | --- |
| 1~5위 | 리라이팅 금지 |
| 6~10위 | 내부링크, CTA, 이미지 설명 보강 |
| 11~20위 | 제목 후보와 도입부 보강 검토 |
| 21위 이하 | 구조 보강 또는 새 각도 리라이팅 |
| TOP 밖 | 방어형 신규 글 또는 전면 리라이팅 후보 |

## 8. 포스팅 후 루프

발행 후 사용자가 URL을 주면 다음 순서로 처리한다.

1. `POSTING_REGISTRY.md` URL 등록
2. 제목이 깨져 있으면 사용자가 준 실제 제목으로 교정
3. `npm run track`
4. `npm run analyze`
5. `npm run dashboard`
6. 바뀐 결과를 기준으로 이 플레이북의 큐를 조정할 필요가 있는지 판단

## 9. 자동 검수 명령

| 명령 | 역할 |
| --- | --- |
| `npm run validate:posts` | 068번 이후 원고 품질 게이트. fail은 발행 전 수정 |
| `npm run validate:posts:strict` | warning까지 실패 처리하는 강한 검수 |
| `npm run validate:data` | config, 순위 이력, 등록부 형식 검증 |
| `npm run check:freshness` | 순위/TOP10/대시보드 최신성 점검 |

| `npm run ops:daily` | `track → dashboard → ranking summary → freshness` 일일 루틴 |
