# AppSheet 현장 스토리 워크플로

> 목적: 통계 기반 블로그 글감이 단순 정보글로 마르지 않도록, 문장군 AppSheet 실제 시공 데이터를 안전하게 연결하는 기준을 정의한다.
> 최종 업데이트: 2026-06-24

## 1. 기본 원칙

문장군 블로그의 경쟁력은 중문 지식만이 아니라 실제 현장 데이터다.

신규 글감은 아래 순서를 기본값으로 삼는다.

```text
통계 검색어
-> 고객 문제 해석
-> 현장 조건/사진 라벨 설계
-> AppSheet 현장 후보 후매칭
-> 필요 시 비식별 evidence_ref 기록
-> 현장 이야기 구성
-> 발행 하드게이트
```

검색어만으로 정보글을 쓰지 않는다. 글마다 최소한 하나의 현장 상황, 구조 변수, 고객 고민, 시공 전후 판단 포인트를 넣는다.

다만 AppSheet 원본을 확인하지 못했거나 근거 ID가 없으면 실제 고객 사례처럼 쓰지 않는다. 이 경우에는 "현장에서 자주 확인되는 조건"처럼 일반화된 표현만 사용한다.

## 2. 글감 선정 시 필수 설계

통계 기반 신규 글감 1개마다 아래 항목을 먼저 정한다. 단, AppSheet 원본 후보를 찾느라 원고 작성이 멈추면 안 된다. 작성자는 현장 조건, 사진으로 보여줄 장면, 고객 고민을 먼저 설계하고, 직원이 발행 작업 때 AppSheet 원본 사진과 사례를 후매칭할 수 있도록 본문 문장 자체를 사진 붙이기 쉬운 흐름으로 쓴다.

| 항목 | 기준 |
| --- | --- |
| 검색어 | 최근 일일 관제 리포트에서 반복된 유입어 |
| 고객 문제 | 검색자가 실제로 겪는 불편, 걱정, 후회 지점 |
| AppSheet 검색 조건 | 제품, 구조, 평수, 지역, 특이사항, 사진 라벨 |
| 현장 후보 | 있으면 비공개 원본을 가리키는 불투명 ID, 없으면 후매칭 필요로 표시 |
| 사진 후보 | 있으면 Before/After 또는 구조 설명에 쓸 비공개 사진 ID, 없으면 필요한 컷을 표시 |
| 공개 가능 범위 | 지역 단위, 평수, 제품명, 구조 변수만 사용. 실명, 주소, 상담 원문 금지 |

예시:

```json
{
  "topic": "신발장 간섭 중문",
  "search_terms": ["신발장 간섭 중문 설치", "좁은 현관 중문", "몰딩있는 현관 중문설치"],
  "appsheet_search": {
    "product": "3연동중문 또는 스윙",
    "structure": "신발장 간섭, 스위치 간섭, 좁은 현관",
    "photo_labels": ["before", "after", "신발장", "가벽", "스위치"]
  },
  "evidence_refs": ["APPSHEET_CASE_OPAQUE_ID"],
  "photo_refs": ["APPSHEET_PHOTO_OPAQUE_ID"],
  "public_scope": "지역/평수/구조 변수만 익명화해 사용"
}
```

## 3. 현장 이야기 구성 규칙

정보글이라도 본문 안에 현장 이야기를 넣는다.

권장 구조:

```text
1. 검색자가 겪는 문제
2. AppSheet에서 매칭했거나 후매칭할 현장 상황
3. 현장에서 판단이 갈린 이유
4. 문장군이 확인한 선택 기준
5. 독자가 자기 집에 적용할 체크포인트
```

현장 이야기에는 아래 중 최소 2개 이상을 포함한다.

- 현관 폭, 벽면, 신발장, 스위치, 턱, 몰딩 등 구조 변수
- 시공 전 고객 고민
- 실측 때 확인한 변수
- 제품 선택이 바뀐 이유
- 사진으로 보여줄 수 있는 Before/After 포인트
- 견적 범위가 달라지는 이유

## 4. 금지 표현

근거가 없으면 아래처럼 쓰지 않는다.

- "안산 고객님은 이렇게 말씀하셨습니다."
- "실제 고객님이 만족하셨습니다."
- "이 현장은 소음이 확실히 줄었습니다."
- "사진처럼 완벽하게 해결됩니다."

근거가 없을 때 허용되는 표현:

- "현장에서는 이런 구조를 먼저 확인합니다."
- "비슷한 구조에서는 신발장 간섭 여부를 먼저 봐야 합니다."
- "실측 전에는 사진만으로 단정하기 어렵습니다."

## 5. 발행 제어 연결

실제 고객 사례, 현장 사례, 사진, 직접 인용, 강한 성능 주장을 쓰는 글은 발행 제어 폴더에 `EVIDENCE.json`을 둔다.

```text
outputs/publish_control/NNN_키워드/
├── STATUS.md
├── APPROVAL_LOG.md
└── EVIDENCE.json
```

공개 저장소에는 AppSheet 원본 레코드, 원본 사진, 고객명, 주소, 상담 원문을 넣지 않는다. 공개 저장소에는 비공개 원본을 가리키는 불투명 `evidence_ref`와 익명화된 요약만 남긴다.

`EVIDENCE.json`의 `evidence_scope`는 본문 주장과 맞아야 한다.

```json
{
  "source_type": "appsheet_case",
  "evidence_refs": ["APPSHEET_CASE_OPAQUE_ID"],
  "photo_refs": ["APPSHEET_PHOTO_OPAQUE_ID"],
  "quote_status": "none",
  "privacy_status": "anonymized",
  "evidence_scope": {
    "region": "수도권",
    "product": "3연동중문",
    "case_type": "shoe_cabinet_interference"
  }
}
```

## 6. 이번 098~103 글감 적용 원칙

다음 글들은 모두 현장 조건형으로 작성하고, 별도 제작노트 없이 본문 문장 안에 AppSheet 후매칭 방향이 드러나게 쓴다. 글 작성자가 AppSheet 원본을 먼저 찾아 작업을 멈추지 않는다. 직원이 실제 AppSheet 사진을 붙일 때 실제 고객 사례 표현으로 바꾸려면 `EVIDENCE.json`을 추가한다.

| 번호 | 글감 | AppSheet에서 찾을 현장 조건 |
| --- | --- | --- |
| 098 | 3연동중문 브론즈유리 | 3연동중문, 브론즈/모루/플루트 유리, 현관 조도·시선 차단 사진 |
| 099 | 중문 셀프시공 | 문장군 완료 현장에서 확인 가능한 수평, 레일, 프레임, 벽면, 신발장 간섭 변수 |
| 100 | 문장군 도어 | 문장군 중문/도어 선택 상담, 제품 범위 설명이 필요한 사례 |
| 101 | 신발장 간섭 중문 | 신발장 돌출, 스위치 간섭, 좁은 현관, 가벽 여부 판단 사례 |
| 102 | 중문 설치 소음 분진 | 거주중 중문시공, 작업 시간, 생활 동선, 소음·분진 준비 사례 |
| 103 | 화장실문 교체 타일 마감 | 화장실문/문틀/타일 경계, 문짝+문틀세트, 문선마감 사례 |

AppSheet 후보가 없는 글은 실제 사례형으로 발행하지 않는다. 이때는 "현장에서 자주 확인되는 조건"으로 작성하고, 필요한 사진 컷과 검색 조건은 별도 지시문이 아니라 본문 문장 흐름으로 유도한다.

## 6. 현장 서사 슬롯

`## 실제 시공 현장에서는 조금 다릅니다` 단락은 AppSheet 후매칭의 중심이다.

이 단락은 아래 5가지를 자연문으로 포함해야 한다.

1. 지역·아파트·구조로 치환 가능한 현장 첫 문장
2. 고객이 처음 망설인 이유
3. 현장에서 확인한 변수
4. 문장군이 판단한 기준과 진행 방향
5. 완료 사진이나 상담 결과로 보여줄 변화

예시:

```markdown
실제로 좁은 현관 폭 때문에 중문 설치를 망설이셨던 현장도 있었습니다.

고객님은 처음에 3연동중문을 생각하고 계셨지만, 현관 가로폭이 좁아 보여서 설치 후 신발 신는 공간이 더 답답해질까 걱정하셨습니다.

현장에서 가로폭과 신발장 끝선, 문짝이 모이는 위치를 함께 확인해보니 단순히 넓고 좁음만 볼 문제는 아니었습니다.

이 현장은 동선을 확인한 뒤 3연동중문 진행 방향을 잡을 수 있었습니다.
```

실제 현장을 매칭한 뒤에는 첫 문장을 `노원구 상계주공아파트 현장도 비슷했습니다`처럼 바꿀 수 있다. 단, 실제 지역·아파트명을 쓰는 경우에는 AppSheet 근거가 있어야 하며 공개 가능한 익명화 범위를 확인한다.

실패 사진이 없을 가능성이 높은 소재는 실패 사례처럼 쓰지 않는다. 예를 들어 셀프중문 글은 "실패한 집"을 꾸미지 않고, 문장군 완료 현장에서 사진으로 확인 가능한 수평, 레일, 프레임, 벽면, 신발장 간섭 같은 변수로 설명한다.
