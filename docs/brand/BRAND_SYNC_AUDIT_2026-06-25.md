# BRAND_SYNC_AUDIT_2026-06-25

> 작성일: 2026-06-25
> 대상: 중앙 브랜드 폴더 `문장군_브랜드` vs 블로그 프로젝트 `docs/strategy/BRAND_CONTEXT.md`
> 목적: 중앙 브랜드 원본과 블로그 브랜드 스냅샷의 차이를 정리하고, 이후 동기화 기준을 고정한다.

## 1. 결론

중앙 브랜드 폴더는 문장군의 기본 정체성, 디자인, 리뷰 표현, 브랜드 톤을 잘 담고 있다.

블로그 프로젝트의 `BRAND_CONTEXT.md`는 2026-06-25 사장 인터뷰를 통해 실제 상담, 실측, 시공 가능 여부, 추가금, 문선, 셀프중문, 거주중 시공, AppSheet 현장 사진 활용 기준이 더 최신이고 깊다.

따라서 이번 동기화 방향은 아래와 같다.

```text
중앙 브랜드 폴더
  브랜드 정체성 + 공통 현장 판단 기준으로 확장

블로그 프로젝트
  중앙 원본을 읽되, 네이버 블로그 운영 어댑터와 발행 하드게이트를 유지
```

## 2. 중앙에 이미 있고 블로그가 참고해야 할 것

| 항목 | 중앙 기준 | 블로그 반영 |
| --- | --- | --- |
| 브랜드 정의 | 무료 방문실측으로 집에 맞는 선택을 돕고 직접 제작과 전속 시공으로 책임지는 브랜드 | 블로그 도입과 CTA에서 직접 활용 |
| 피해야 할 인상 | 싸구려 할인 전단지, 차가운 대기업, AI 문구 | 원고 문체 검수 기준으로 유지 |
| 리뷰 표현 | 전체 상품 리뷰 3만+, 대표상품 단일 리뷰 1.4만+ | 기존 15,000개 표현은 전체 브랜드 리뷰처럼 쓰지 않도록 정정 필요 |
| 디자인 기준 | 밝은 주거 공간, 실제 시공 사진, 따뜻한 전문성 | 블로그 썸네일/사진 선정 기준으로 참고 |
| 중앙 적용 방식 | 프로젝트별 보조 규칙은 분리 | 블로그 어댑터 문서로 분리 완료 |

## 3. 블로그에 있고 중앙으로 승격한 것

아래는 블로그 v4.0 인터뷰 기반 정보 중 모든 문장군 프로젝트가 같이 써야 하는 내용이다. 중앙 `FIELD_JUDGMENT_RULES.md`로 승격했다.

- 대표상품가를 최종 시공비로 오해하는 고객 문제
- 전화 확정 견적보다 무료 방문 실측이 필요한 이유
- 타업체 디자인/용어 혼동 대응
- 중문 자재 판매 미진행 기준
- 3연동중문, 원슬라이딩, 스윙의 현장 가능 범위
- 중문틀이 세워질 벽공간과 신발장 간섭 기준
- 기둥, 바닥보강, 스위치등박스, 신발장리폼 등 추가 구조 변수
- 상담 전 사진 요청 기준
- 12mm 슬림문선 기준과 9mm 문선 비교 원칙
- 문틀만 단독 교체 불가, 공틀/쌈지 마감 가능 기준
- ABS도어 문짝만 교체 vs 문짝+문틀세트 차이
- 거주중 시공의 먼지, 분진, 소음, 준비물 기준
- AppSheet 현장 사진을 콘텐츠에 활용하는 기준
- 카카오톡 알림톡, 견적서 알림톡, 시공 발주/배정/완료 안내 같은 운영 신뢰 근거

## 4. 블로그에만 남길 것

아래는 중앙 브랜드 원본이 아니라 블로그 어댑터에 남긴다.

- 네이버 블로그 발행용 단일 MD 원칙
- 제작노트 미사용
- 본문 내 `[사진:]`, `[AppSheet 확인]`, `[제작자 메모]` 금지
- 공백 제외 1,500~2,500자 합격 범위
- 일일 SEO 관제와 게시글 TOP 20 기준
- `ranking_report.md` experimental 격하
- 해시태그는 본문이 아니라 네이버 태그란에 입력
- 블로그 발행 하드게이트
- `## 실제 시공 현장에서는 조금 다릅니다` 필수 단락
- POSTING_REGISTRY 기반 중복 관리

| 항목 | 중앙 v1.0 | 블로그 v4.0/운영 판단 | 처리 |

| 기타 마감 소재 | 중앙은 천정몰딩/걸레받이 프로젝트별 확인 | 블로그에서는 검색어 대응 정보성 소재로 다룸 | 문장군 운영 범위와 글 목적 확인 후 정보성으로만 사용 |

## 6. 앞으로의 동기화 방식

1. 사장 인터뷰나 현장 운영 기준이 새로 나오면 먼저 블로그에 임시 반영하지 않는다.
2. 공통 브랜드 지식인지, 블로그 전용 규칙인지 분류한다.
3. 공통 지식이면 중앙 브랜드 폴더의 `FIELD_JUDGMENT_RULES.md` 또는 `BRAND_CONTEXT.md`에 반영한다.
4. 블로그 전용이면 `BLOG_BRAND_ADAPTER.md`, `AGENTS.md`, 운영 문서에 반영한다.
5. 충돌은 `BRAND_SYNC_AUDIT_YYYY-MM-DD.md` 형식으로 남긴다.
6. 원본 인터뷰, AppSheet 자료, 네이버 통계 원본은 공개 문서에 넣지 않는다.
