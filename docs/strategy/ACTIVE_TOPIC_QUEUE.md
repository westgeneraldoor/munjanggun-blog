# 문장군 Active Topic Queue

> 업데이트: 2026-06-29
> 목표: daily report를 단순 기록으로 끝내지 않고, 일 2,000 조회수 목표를 향해 성과 방어, 신규 공격, 롱테일 실험, 위험 제외를 동시에 운영한다.

## 역할

- daily report는 증거 로그다. 조회수, 유입어, TOP20, 작성일, 다음 액션을 보존한다.
- Active Topic Queue는 실행판이다. daily의 다음 액션은 이 Q-ID로 닫혀야 한다.
- topic scorecard는 신규/공격 후보를 발행 후보로 승격할지 판단하는 근거다.
- Posting Registry는 발행 URL, 작성일, 중복/카니발 방지 원장이다.

## lane/status 계약

| 구분 | 허용값 | 의미 |
| --- | --- | --- |
| lane | protect | 이미 성과가 있는 키워드와 글을 보호하고 내부링크, 후속 글, 제목 후보만 보강 |
| lane | attack | 시장 수요가 크지만 현재 블로그 성과가 약한 키워드 공격 |
| lane | experiment | 검색량은 작아도 전환 의도, 하자, 견적, 현장 불안이 강한 롱테일 실험 |
| lane | exclude | 문장군 미취급, 오해 유발, 서비스 범위 밖 키워드 확장 차단 |
| status | internal_link | 신규 글보다 기존 글 연결 보강이 우선 |
| status | rewrite_candidate | 기존 글 리라이팅 또는 허브 보강 후보 |
| status | scorecard_needed | scorecard 작성 후 원고화 여부 판단 |
| status | draft_ready | 원고 작성 가능 상태 |
| status | publish_waiting | 원고 작성 후 발행/URL 등록/관찰 대기 |
| status | monitor_3d | 3일 이상 반복 유입을 더 봐야 하는 관찰 항목 |
| status | monitor_7d | 7일 이상 반복성, 전환성, 중복 위험을 봐야 하는 관찰 항목 |
| status | excluded | 확장 금지 상태 |
| status | done | 처리 완료 |

## 운영 규칙

- `lane`은 전략 전선이고 `status`는 처리 상태다. `publish_waiting`은 lane이 아니다.
- protect만 늘어나면 조회수 방어는 되지만 일 2,000을 뚫기 어렵다. attack lane은 최소 3개 이상 유지한다.
- attack 후보는 `market_volume`을 비워두지 않는다. 시장 수요가 있어야 공격 전선이다.
- exclude lane은 글감으로 승격하지 않는다. scorecard, draft, publish 액션을 걸지 않는다.
- daily report 마지막의 `오늘 보드 반영`은 아래 Q-ID 중 하나로 닫는다.
- 이 표는 8~15개 실행 항목만 유지한다. 너무 많은 후보는 daily report와 scorecard를 흐린다.
