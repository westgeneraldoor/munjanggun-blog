# 문장군 Active Topic Queue

> 업데이트일: 2026-06-29  
> 목표: 일일 통계 보고를 실행 상태로 닫아 일 2,000 조회수 목표를 향한 소재 보호, 보강, 신규 발굴, 제외 판단을 관리한다.

## 역할

- daily report는 증거 로그다. 조회수, 유입어, TOP20, 작성일, 다음 액션을 보존한다.
- topic scorecard는 신규 또는 공격 후보를 발행 후보로 승격할 때 쓰는 근거다.
- Active Topic Queue는 실행 상태판이다. daily의 다음 액션은 기존 행 갱신 또는 신규 행 추가로 닫는다.
- Posting Registry는 발행 URL, 작성일, 중복/카니발 방지 원장이다.

## 상태값

| status | 의미 |
| --- | --- |
| protect | 이미 성과가 있는 글/키워드를 보호하고 내부링크나 소제목만 보강 |
| publish_waiting | 원고 또는 scorecard가 준비되어 발행/URL 등록 대기 |
| scorecard_needed | 신규/공격 후보라 scorecard 작성 후 발행 여부 판단 |
| draft_ready | 원고 작성 가능 상태 |
| monitor_3d | 3일 이상 반응을 더 봐야 하는 관찰 항목 |
| rewrite_candidate | 기존 글 리라이팅 후보 |
| internal_link | 신규 글보다 내부 연결 보강이 우선 |
| excluded | 문장군 미취급, 오해 유발, 또는 현재 발행 제외 |

## 운영 규칙

- 최근 성과 키워드만 반복하지 않는다. 방어 키워드와 공격 키워드를 별도로 본다.
- 공격 키워드는 시장 수요가 있고, 현재 블로그 성과가 약하며, 문장군 취급 가능하고, AppSheet 후매칭 가능할 때만 후보로 둔다.
- daily에서 나온 다음 액션은 이 queue에 없으면 누락으로 본다.
- 3일 반복 유입은 `monitor_3d`, 7일 반복 유입은 `protect` 또는 `scorecard_needed`로 승격한다.
- 제외 키워드는 삭제하지 않고 `excluded`로 남겨 재등장을 막는다.
- 이 파일은 10~15개 핵심 항목만 유지한다. 세부 원문 통계는 daily report에 남긴다.
