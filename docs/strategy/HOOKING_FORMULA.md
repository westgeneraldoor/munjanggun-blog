# 문장군 블로그 후킹 공식

> 목적: 제목, 도입부 첫 문장, 썸네일 카피에서 검색자가 멈칫하게 만드는 실무형 후킹 기준을 남긴다.
> 최종 업데이트: 2026-06-19

> 상위 원칙: `docs/strategy/CONTENT_OPERATING_PRINCIPLES.md`

이 문서는 문장군 콘텐츠 운영 원칙의 하위 전술 문서다. 후킹은 조회수를 위한 자극 문구가 아니라, 고객이 자신의 문제를 발견하게 만드는 첫 장치로 사용한다.

## 1. 핵심 원칙

후킹은 자극적인 문장을 쓰는 일이 아니다.

문장군 블로그에서 후킹은 검색자가 이미 가진 불안, 오해, 손실 걱정을 정확히 짚고, 글을 읽으면 어떤 기준을 얻을 수 있는지 보여주는 일이다.

```text
좋은 후킹 =
검색자가 가진 고민
+ 흔한 오해 반박
+ 현장에서 확인할 기준
```

## 2. 6가지 후킹 패턴

| 패턴 | 의미 | 문장군식 사용법 |
| --- | --- | --- |
| 호기심 결핍 | 일부 정보를 비워 궁금하게 만든다 | `비용이 달라지는 지점은 따로 있습니다` |
| 구체적 숫자 | 막연한 주장 대신 숫자로 구조를 만든다 | `3가지 기준`, `5종 비교`, `24평/32평` |
| 타깃 호명 | 특정 검색자를 직접 부른다 | `구축 아파트`, `좁은 현관`, `24평 아파트` |
| 통념 반박 | 흔한 믿음을 뒤집는다 | `평수보다 현관 구조가 먼저입니다` |
| 손실 회피 | 놓치면 손해 보는 지점을 보여준다 | `설치 전 이 기준을 놓치면 견적이 흔들립니다` |
| 결과 약속 | 읽은 뒤 얻을 판단 기준을 제시한다 | `우리 집에 맞는 중문 구조를 고르는 기준` |

## 3. 문장군 우선 조합

### 1순위: 통념 반박 + 현장 변수

문장군은 실제 시공 범위와 현장 구조 변수가 중요하다. 그래서 단순 정보보다 오해를 바로잡는 제목이 잘 맞는다.

예시:

- 중문 비용은 평수보다 현관 구조에서 갈립니다
- 문틀교체비용은 문틀 하나 가격이 아닙니다
- PVC 걸레받이 단점은 자재보다 마감에서 보입니다

### 2순위: 손실 회피 + 체크 기준

강한 공포 표현보다, 놓치면 견적이나 마감이 흔들리는 기준을 알려주는 방식으로 쓴다.

예시:

- 중문 설치 전 놓치기 쉬운 3가지 구조 기준
- 방문교체비용이 달라지는 3가지 범위
- 걸레받이 교체 전 먼저 볼 3가지 마감 기준

### 3순위: 타깃 호명 + 문제

평수, 구축, 좁은 현관, 거주중 시공처럼 검색자가 자기 상황을 바로 알아보게 만든다.

예시:

- 24평 아파트 중문 고민 전 확인할 3가지
- 구축 아파트 방문교체 전 먼저 볼 3가지
- 좁은 현관 스윙중문 선택 전 보는 3가지

## 4. 실무 공식

### 공식 A: 타깃 + 문제 + 반전

```text
[타깃] + [문제] + [흔한 오해 반박]
```

예시:

- 평수별 중문 비용, 평수보다 먼저 볼 3가지
- 좁은 현관 중문, 종류보다 열림 반경이 먼저입니다
- 구축 아파트 문교체비용, 문짝만 보면 견적이 흔들립니다

### 공식 B: 손실 + 호기심

```text
[놓치기 쉬운 문제] + [이유를 비워둔 표현]
```

예시:

- 중문 설치 전 이 기준 놓치면 추가 비용이 생깁니다
- PVC 걸레받이 고르기 전 꼭 봐야 할 3가지
- 9mm 문선 선택 전 관리 기준을 먼저 봐야 합니다

### 공식 C: 통념 반박

```text
[흔한 믿음] + [아니다/먼저 볼 것은 따로 있다]
```

예시:

- 비싼 중문이 좋은 중문은 아닙니다
- 걸레받이는 방수성만 보고 고르면 안 됩니다
- 문틀교체는 문틀만의 문제가 아닙니다

### 공식 D: 숫자 + 결과

```text
[숫자] + [독자가 얻는 판단 결과]
```

예시:

- 현관 구조 3가지로 중문 견적이 달라집니다
- 문선 2가지 차이로 마감 느낌이 달라집니다
- 중문 종류 5종을 1분 만에 고르는 기준

## 5. 감사원 게이트 기준

후킹을 쓰더라도 아래는 금지한다.

| 금지 표현 | 이유 |
| --- | --- |
| 근거 없는 `40% 감소`, `90% 차단`, `100% 해결` | 수치/성능 주장 근거 필요 |
| 실제 고객 사례처럼 보이지만 `evidence_ref` 없는 표현 | P1 Evidence Gate 차단 대상 |
| 직접 인용처럼 보이는 따옴표 문장 | `quote_status` 없으면 차단 대상 |
| `완벽`, `확실`, `무조건`, `보장` | 강한 보장/확신 표현 |
| 문장군 미취급 제품을 가능한 것처럼 쓰는 후킹 | 제품 검증 게이트 차단 대상 |
| 현관문, 방화문, 중문파티션 확장 후킹 | 제외/주의 키워드 |

## 6. 제목 적용 체크

새 글 제목을 정할 때 아래 순서로 점검한다.

1. 타겟 키워드가 앞쪽에 있는가.
2. 숫자 또는 구체적 기준이 있는가.
3. 검색자의 오해나 손실 걱정을 건드리는가.
4. 과장된 성능, 보장, 실제 사례처럼 보이는 표현은 없는가.
5. 기존 보호 글과 같은 각도를 반복하지 않는가.

## 7. 문장군식 예시

| 약한 제목 | 개선 방향 |
| --- | --- |
| 평수별 아파트 중문 설치 비용 3가지 | 평수별 중문 비용, 평수보다 먼저 볼 3가지 |
| PVC 걸레받이 단점 3가지 | PVC 걸레받이 단점, 자재보다 마감이 문제입니다 |
| 스윙중문 선택 기준 | 스윙중문 단점, 좁은 현관이면 기준이 다릅니다 |
| 문교체비용 견적 기준 | 문교체비용, 문짝만 보면 견적이 흔들립니다 |
| 9mm 문선 관리 방법 | 9mm 문선, 얇아 보이는 것보다 중요한 3가지 |

## 8. 운영 메모

- 정보글은 자극보다 신뢰를 우선한다.
- 비용형 글은 `손실 회피 + 체크 기준`이 잘 맞는다.
- 제품 비교 글은 `통념 반박 + 현장 변수`가 잘 맞는다.
- 마감재 글은 `단점 + 관리/마감 기준`이 잘 맞는다.
- 실제 사례형 후킹은 근거 파일이 준비된 경우에만 사용한다.
